| 任务 | 核心内容 | 关键算法 |

## 三、开发步骤（按实现顺序）

下面是我实际开发时的步骤，你可以按这个顺序理解或修改代码：

### 步骤1：搭建框架

1. 创建 `memory_management.c`
2. 写好头文件引用（`stdio.h`, `stdlib.h`, `string.h`, `time.h`）
3. 定义常量（`MAX_BLOCKS=100`, `TOTAL_MEMORY=640`, `MAX_PAGES=1200`, `MAX_FRAMES=20`）
4. 加 `_CRT_SECURE_NO_WARNINGS` 宏（VS2022 兼容）
5. 加 `#ifdef _WIN32` 的 `windows.h`（控制台 UTF-8 支持）

### 步骤2：实现任务1 — 数据结构

1. 定义 `MemoryBlock` 结构体：`start_addr`, `size`, `status`, `process`
2. 声明全局数组 `frees[MAX_BLOCKS]` 和 `occupys[MAX_BLOCKS]`
3. 声明计数器 `free_count`, `alloc_count`

### 步骤3：实现任务1 — 辅助函数

1. **`sort_free_table()`**：冒泡排序，按 `start_addr` 升序
2. **`merge_free_blocks()`**：
   - 先调 `sort_free_table()`
   - 遍历相邻块，如果 `frees[i].start_addr + frees[i].size == frees[i+1].start_addr`，则合并
   - 合并方式：`frees[i].size += frees[i+1].size`，然后删除 `frees[i+1]`
3. **`print_fragmentation()`**：
   - 遍历 frees[] 求总空闲和最大连续块
   - `外部碎片 = 总空闲 - 最大连续块`
   - `碎片率 = 外部碎片 / TOTAL_MEMORY * 100%`
4. **`print_memory_status()`**：格式化打印两张表 + 调 `print_fragmentation()`

### 步骤4：实现任务1 — 核心算法

1. **`allocate_memory(process_name, req_size)`**：
   - 调 `sort_free_table()` 保证从低地址开始
   - 遍历 frees[]，找第一个 `size >= req_size` 的块
   - 如果恰好相等：删除该空闲块
   - 如果更大：分割（`start_addr += req_size`, `size -= req_size`）
   - 写入 occupys[]
   - 返回起始地址（失败返回 -1）

2. **`free_memory(process_name)`**：
   - 在 occupys[] 中找到该进程
   - 将其区域加入 frees[]
   - 从 occupys[] 中删除
   - 调 `merge_free_blocks()` 合并相邻块

3. **`task1_dynamic_partition()`**：按 9 步脚本依次调用

### 步骤5：实现任务2 — FIFO 算法

```c
int fifo_replace(int seq[], int seq_len, int num_frames, int verbose)
```

核心逻辑：
- `frames[MAX_FRAMES]` 数组表示物理帧，初始化为 -1
- `oldest` 指针指向最早进入的帧位置（循环递增）
- `count` 记录当前已用帧数
- 每次访问页面：
  1. 先检查是否命中（遍历 frames[]）
  2. 命中则跳过
  3. 未命中（缺页）：
     - 如果帧没满：放入下一个空位，`count++`
     - 如果帧满了：替换 `frames[oldest]`，`oldest = (oldest+1) % num_frames`

### 步骤6：实现任务2 — LRU 算法

```c
int lru_replace(int seq[], int seq_len, int num_frames, int verbose)
```

核心逻辑：
- `frames[MAX_FRAMES]` 和 `last_used[MAX_FRAMES]` 两个数组
- `last_used[i]` 记录帧 i 最后被访问的时间（用循环计数器 step）
- 每次访问页面：
  1. 检查是否命中，命中则更新 `last_used[hit_pos] = step`
  2. 未命中（缺页）：
     - 帧没满：放入空位
     - 帧满了：找 `last_used[]` 最小的帧，替换之

### 步骤7：实现任务2 — 测试流程

`task2_page_replacement()` 按以下顺序执行：

1. **固定序列测试**：`[1,2,3,4,1,2,5,1,2,3,4,5]`，3帧 和 4帧，verbose=1 打印每步
2. **Belady 异常输出**：对比 FIFO 3帧(9次) vs 4帧(10次)
3. **随机序列测试**：生成 1000 个随机页面引用（0~9），帧数 1→10，输出 CSV
4. **局部性序列测试**：用 `generate_locality_sequence()` 生成序列，帧数 1→10，输出 CSV
5. **Belady 异常专项验证**：帧数 1→6，对比 FIFO 和 LRU，输出 CSV

### 步骤8：CSV 输出

每组测试中：
1. `fopen("xxx.csv", "w")` 打开文件
2. `write_utf8_bom(fp)` 写 UTF-8 BOM（3 字节：`0xEF 0xBB 0xBF`）
3. `fprintf(fp, "表头\n")` 写 CSV 表头
4. 循环中 `fprintf(fp, "%d,%d,%.1f\n", ...)` 写数据行
5. `fclose(fp)` 关闭文件

### 步骤9：main() 函数

```c
int main(void) {
    // Windows 下设置控制台 UTF-8
    SetConsoleOutputCP(65001);
    // 打印标题
    // 执行任务1
    task1_dynamic_partition();
    // 执行任务2
    task2_page_replacement();
    // 打印结束信息
    return 0;
}
```

### 5.2 添加新算法（例如最佳适应 Best Fit）

如果想添加新的内存分配算法：

1. 复制 `allocate_memory()` 函数，改名为 `allocate_memory_best_fit()`
2. 把循环逻辑从"找到第一个就返回"改为"遍历所有，找最小的够用块"：
   ```c
   // 首次适应（现有代码）：
   if (frees[i].size >= req_size) { return ...; }  // 找到就用

   // 最佳适应（修改版）：
   int best = -1;
   for (i = 0; i < free_count; i++) {
       if (frees[i].size >= req_size) {
           if (best == -1 || frees[i].size < frees[best].size)
               best = i;
       }
   }
   // 用 frees[best] 分配
   ```

### 5.3 添加新的页面置换算法（例如 OPT 最优置换）

1. 复制 `fifo_replace()` 函数，改名为 `opt_replace()`
2. 淘汰策略改为"向后看未来序列，找将来最久不被访问的页面"：
   ```c
   // 找到要淘汰的帧
   int victim = 0, farthest = -1;
   for (j = 0; j < num_frames; j++) {
       int next_use = seq_len;  // 默认永远不再使用
       for (k = i + 1; k < seq_len; k++) {
           if (seq[k] == frames[j]) { next_use = k; break; }
       }
       if (next_use > farthest) { farthest = next_use; victim = j; }
   }
   frames[victim] = seq[i];
   ```

## 六、关键知识点回顾

### 首次适应算法（First Fit）
- 从空闲表**最低地址**开始扫描，找到**第一个**够大的块就分配
- 优点：简单快速
- 缺点：低地址区容易积累小碎片

### FIFO 页面置换
- 先进先出，淘汰**最早进入**内存的页面
- 用一个 `oldest` 指针（循环队列头）实现
- 缺点：可能出现 **Belady 异常**（增加帧数反而缺页更多）

### LRU 页面置换
- 淘汰**最近最久未使用**的页面
- 用 `last_used[]` 时间戳数组实现
- 优点：利用时间局部性，不会出现 Belady 异常（栈算法）
- 缺点：实现开销大，需要硬件支持或近似算法

### Belady 异常
- 本实验中的证明：序列 `[1,2,3,4,1,2,5,1,2,3,4,5]`
- FIFO 3帧 → 9次缺页，4帧 → **10次缺页**（增加了！）
- 原因：FIFO 不是栈算法，不满足包含性质

### 外部碎片率
- `外部碎片 = 总空闲内存 - 最大连续空闲块`
- `碎片率 = 外部碎片 / 总内存 × 100%`
- 碎片率高说明虽然总空闲够，但被分成了很多小块，无法分配大请求
